# System patterns

- **Parsing:** Custom lexer in `lexer.rs` emits `FuncIdent(name)` when identifier is followed by `(`, else `Ident(name)`; disambiguates `sin(1)` (call) vs `sin` (symbol). Keywords: "per" (division), "as" (unit conversion; boundary check so "aspect" stays Ident). Comparison tokens: `==`, `!=`, `<`, `<=`, `>`, `>=` (two-char before single `<`/`>`). LALRPOP grammar in `expr.lalrpop` uses extern tokens (Num, Ident, FuncIdent, etc.). Precedence: top-level Expr (comparison: ==, !=, <, <=, >, >=) ← AddExpr (+, -, as) ← Term (*, /, per, implicit mul) ← Factor (unary minus, NumIdents, literal, `FuncIdent "(" ArgList ")"`, Ident, parentheses). **UnitTerm:** unit-only expression (IdentIdents, *, /, per) for RHS of "as" (e.g. "m", "meters per second"). **NumIdents:** number followed by one or more idents is one factor (e.g. "1 s", "2 pi rad"); parses as left-associated Mul chain so " / 1 s" divides by 1 s and "2 pi rad" has pi→symbol, rad→unit at resolve. **IdentIdents:** one or more idents with no number (e.g. "pi", "pi rad") is one factor; "pi rad" parses as Mul(pi, rad), resolve attributes to symbol and unit. **PiIdents:** "π" alone or "π" + one or more idents (e.g. "π rad") so unicode π behaves like "pi rad". Implicit mul RHS is only number or parenthesized expr (not ident). Call args: `CallArg` = `Ident ":" Expr` (named) or `Expr` (positional); `ArgList` left-recursive.
- **IR:** `ExprDef` has parse-time variants `LitScalar`, `LitWithUnit`, `LitUnit`, `Neg(Box<ExprDef>)`, `VecLiteral(Vec<ExprDef>)`, `Transpose(Box<ExprDef>)` and resolved `Lit(Quantity)`, `LitFuzzyBool(FuzzyBool)` (constant-folded comparison), `LitSymbol(String)`, `Call(String, Vec<CallArg>)`, `As(Box<ExprDef>, Box<ExprDef>)` (unit conversion: left value expr, right unit-only expr). Binary `Add`, `Sub`, `Mul`, `Div`. Comparison: `Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two `Box<ExprDef>`); **result is Value::FuzzyBool**. `CallArg` = `Positional(Box<ExprDef>)` or `Named(String, Box<ExprDef>)`. Two-step flow: parse → resolve (with unit registry) → `ExprDef` with `Lit(Quantity)`, `LitFuzzyBool`, `LitSymbol`, `Call`, `As`, or `VecLiteral` for literals. RHS of `As` resolved via `resolve_unit_expr` (only LitUnit/Mul/Div → Lit(Quantity), no symbols). `ExprData` has `LitFuzzyBool(FuzzyBool)`, `Call(name, Vec<(Option<String>, Expression)>)`, `As(Expression, Expression)`, `VecLiteral(Vec<Expression>)`, and `Transpose(Expression)`; `value()` returns `Result<Value, RunError>`.
- **Numbers and quantities:** Numeric values are `SnaqNumber` (value + variance). Variance propagates through arithmetic (Add/Sub: sum of variances; Mul/Div: standard propagation; scale by f64: variance × factor²). Literals get variance from f64 precision via `SnaqNumber::from_literal`. `Quantity` holds `number: SnaqNumber` and unit; `new(f64, unit)` / `from_scalar(f64)` use `from_literal`; results of arithmetic use `with_number(SnaqNumber, unit)`.
- **Units:** Default registry (`default_si_registry`) provides all 7 SI base units (m, kg, s, A, K, mol, cd) and all SI derived units with special names (J, C, V, F, ohm, S, Wb, T, H, Hz, N, Pa, W, lm, lx, Bq, Gy, Sv, kat), plus km, g, hour, minute, second, day, week, month, quarter, year, decade, century, millennium, mile, au, parsec, light_year, joule, eV, celsius, Angle (rad base, degree = π/180 rad), and **Area** (m2, are = 100 m², hectare = 100 are; hectare via prefix "hect" + "are"). Squared-length: prefix + "m2" or "m²" in `try_lookup` resolves to (prefix·m)² so km2, cm2, mm2 convert correctly. Area aliases: m², sqm, squaremeter, squarekilometer, etc.; inch (in), in2, in², sqin, squareinch; foot (ft), ft2, sqft, squarefoot, square_foot. Registry stores singular forms; plural input normalized via `singularize`. Long-form aliases via `add_derived_alias`. Prefix resolution via `get_unit_with_prefix`. Quantity has `full_simplify` and `full_simplify_with_registry`.
- **Errors:** LALRPOP `ParseError` is mapped to `crate::error::ParseError` in the parser wrapper. Generated code has targeted `#[allow(...)]` for Clippy. Transpose on non-vector → `RunError::ExpectedVector`. Vector op length mismatch (element-wise or dot) → `RunError::VectorLengthMismatch { left_len, right_len }`.
- **±∞:** Only +∞ and -∞ supported (no NaN). Division: nonzero/0 → ±∞, 0/0 → DivisionByZero. Infinite values have variance 0; display and arithmetic via f64. `Quantity::is_infinite()` / `is_finite()`; `convert_to` short-circuits when value is infinite (unit change only).
- **Value and symbols:** `Value` is `Numeric(Quantity)`, `Symbolic(SymbolicQuantity)`, **FuzzyBool(FuzzyBool)** (comparison result: True, False, or Uncertain(prob)), or `Vector(VectorValue)`. Display: Numeric/Symbolic as before; FuzzyBool as "true"/"false"/"uncertain(prob)". `to_quantity` on FuzzyBool or on Symbolic with comparison expr → `RunError::BooleanResult`. Comparisons use statistical P(A>B) and CONFIDENCE_THRESHOLD (0.95); when variance is zero or means equal (for Ge/Le) result is crisp. `SymbolRegistry` maps symbol names to numeric values (built-ins: pi, π, e, sqrt_2, sqrt_3). Unknown identifiers resolve to `LitSymbol`. Evaluation returns `Value`. `SymbolicExpr` (Number, Symbol, Sum, Product, Neg, Div, Call, Eq, Ne, Lt, Le, Gt, Ge); `SymbolicQuantity(expr, unit)`. Comparison expressions: numeric comparison → `Value::FuzzyBool`; symbolic comparison stays Symbolic; `expr.is_comparison()` used to reject `to_quantity`. Default: symbolic result; `run_numeric()` on comparison returns `BooleanResult`. CLI: default symbolic, `--numeric` for numeric. Error `SymbolHasNoValue(name)` when substituting and symbol has no value.
- **Functions:** Built-ins in `functions.rs`: sin, cos, tan (one arg, Angle dimension: rad or degree; converted to radians internally); max, min (two args, same dimension). For well-known angles (0, π/6, π/4, π/3, π/2, π and degree 0°, 30°, 45°, 60°, 90°, 180°), trig returns symbolic exact results (e.g. √2/2, √3/2 via constants sqrt_2, sqrt_3). `KnownAngle`, `known_angle_from_rad`, `known_angle_from_symbolic`, `exact_sin/cos/tan`; eval_call uses them so symbolic run keeps constants (π, √2, √3) in display and run_numeric yields a number. Each built-in declares param names; call-site args bound positionally then by name. Unknown function → `RunError::UnknownFunction`. Wrong dimension for trig → `RunError::ExpectedAngle { actual }`. Symbolic calls: if any arg is symbolic and not a known angle, result is `Value::Symbolic(SymbolicExpr::Call(name, args))`. Variance is propagated through trig via first-order propagation.
- **CAS:** `cas/` module: interned AST (`ExprId`, `ExprNode`, `ExprInterner`), canonicalization (flatten Add/Mul, sort by rank), rewrite (identity/annihilation, constant fold, like-term collection). **LitFuzzyBool:** `ExprNode::LitFuzzyBool(FuzzyBool)`; rewrite constant-folds comparisons using statistical P(A>B) and CONFIDENCE_THRESHOLD to `LitFuzzyBool(True/False/Uncertain(prob))`. **Comparison nodes:** `ExprNode::Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two ExprId); pass-through in canonicalization; rewrite when both children Lit → `LitFuzzyBool`. **Mul operands:** sorted by rank except when both operands are vector-valued (`VecLiteral` or `Transpose`): then order is preserved so row×column (dot) vs column×row (outer) is not swapped. `ExprNode::VecLiteral(Vec<ExprId>)` pass-through. `simplify_symbolic(def, registry)` and `simplify_numeric(def, unit_registry, symbol_registry)`; `substitute_symbols(def, symbol_registry)` handles `LitFuzzyBool`. Run pipelines apply CAS before building Salsa program. Division by zero during rewrite returns `RunError::DivisionByZero`.
- **Vectors:** `Value::Vector(VectorValue)` where `VectorValue { inner: LazyVector, orientation: VectorOrientation }`. Orientation: **Column** (default for literals and 2D column yields), **Row** (after transpose); transpose flips orientation. Lazy vector: `LazyVector::FromExprs`, `FromEvaluated`, `Map`, `ZipMap`, `Outer`, stub `Transform`, and **Transpose** (postfix `'`): 1D stream = identity; 2D = `VectorStream::Transposed` yields columns. **Vector–vector ops** (add, sub, mul, div, and comparison ==, !=, <, <=, >, >=) dispatch by orientation: **column×column** or **row×row** → element-wise (lazy `ZipMap`), result column or row (comparisons yield FuzzyBool per element); **column×row** → outer product (lazy `Outer`), result = vector of column vectors (matrix of FuzzyBool for comparisons); **row×column** → dot product (mul), sum(left) op sum(right) (add/sub), or compare(sum(left), sum(right)) (comparisons), scalar. `VectorBinaryOp` (Add, Sub, Mul, Div, Eq, Ne, Lt, Le, Gt, Ge) and `VectorMapOp` (including Eq/Ne/Lt/Le/Gt/Ge(Box<Value>) for vector op scalar) used by ZipMap, Outer, and Map. Streams: `ZippedVectorStream`, `OuterProductVectorStream`. **Implicit mapping:** vector ±/×/÷ scalar → Map; vector op scalar comparison → Map (0/1 per element); neg/sin/cos/tan(vector) → Map. `vector_into_stream` handles Map, ZipMap, Outer; op applied via `apply_map_op` / `apply_binary_op`. to_quantity on vector → `UnsupportedVectorOperation`. Literal syntax `[ expr, ... ]` or `[]`; postfix `'` on any Factor. WASM-compatible (futures only in lang crate).
