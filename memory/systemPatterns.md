# System patterns

- **Input declarations:** `input` declarations are function-like scoped bindings. Duplicate input names inside the same block are rejected at runtime with `RunError::InvalidArgument("duplicate input declaration: <name>")` to keep graph param wiring deterministic.
- **Graph rename refactor policy:** `snaqlite/graph/renameParam` updates both the declaration and safe in-scope identifier usages that resolve to that input; usages under shadowing bindings/params are intentionally preserved.
- **Stable graph input identity:** input declarations support `input name@paramId: Type`. Runtime scope binding still uses `name`; graph wiring and reconciliation use stable `paramId` so UI renames do not force disconnect/reconnect.
- **LSP node signature payload:** `snaqlite/graph/nodeSignatureUpdated.inputs` now includes `{ name, paramId, type }`.
- **LSP connect target matching:** `graph/connect.targetInputName` accepts either display `name` or stable `paramId`; edges are stored by `paramId`.
- **LSP disconnect target matching:** `graph/disconnect.targetInputName` must resolve using the same display `name`/`paramId` lookup as connect before mutating graph state; disconnecting by display name should remove edges keyed by stable `paramId`.
- **Target input resolution precedence:** when matching `targetInputName`, prefer exact `paramId` match before display-name match to avoid ambiguous collisions (e.g. one input name equals another input `paramId`).
- **Node-centric subscription contract:** `snaqlite/subscribeNode` / `snaqlite/unsubscribeNode` are canonical APIs; `snaqlite/subscribe` / `snaqlite/unsubscribe` remain compatibility aliases over the same subscription runtime.
- **WASM vector running batches:** WASM node/document vector subscriptions emit progressive `Running` payload batches (using the same payload shape as native) before terminal `Completed`.
- **Graph-aware node subscribe execution:** `subscribeNode` runs through graph runtime (`run_node_with_graph_inputs`) so subscriptions on wired downstream nodes evaluate with upstream inputs, unlike legacy root subscribe behavior.
- **Dual publish notification compatibility:** LSP emits both `snaqlite/publishResult` and `snaqlite/publishNodeResult` with identical payloads so new and legacy clients remain in sync during migration.
- **Canonical canvas snapshot transport:** LSP exposes `snaqlite/graph/exportCanvasDocument` / `importCanvasDocument` using language-level canvas model shape (`canvasId`, `nodes[{uri,source,version}]`, `edges[{sourceUri,targetUri,targetParamId}]`) for recoverable runtime state.
- **Canvas import lifecycle:** importing a canvas is destructive replacement of runtime graph state: cancel all active subscriptions/widgets (`reason: "Canvas import"`), clear node-result cache/documents, replace edges/documents from snapshot, then recompute imported nodes and republish diagnostics/signatures.
- **Canvas import validation:** imported canvas snapshots must not contain duplicate node URIs; import rejects duplicates with `invalid_params`.
- **Cancellation fanout for drained registries:** destructive lifecycle paths (shutdown/import) must emit terminal cancellation notifications for all removed subscription/widget IDs, including entries without active consumer senders.
- **Hard canvas isolation:** LSP binds to one canvas identity on first URI usage (`snaq://` host for graph URIs) and rejects operations from mismatched canvases.
- **Cycle prevention on connect:** graph connect performs pre-mutation cycle detection and rejects cycle-creating edges with server error `-32002`.
- **Canvas guard for slice fetch:** `snaqlite/graph/fetchResultSlice` validates the widget's `source_uri` with the same hard canvas binding used by other graph methods, rejecting cross-canvas slice reads.
- **Graph-aware connect type inference:** source output type checks during connect use full graph execution for the source node (upstream-aware), not empty-input single-node execution.
- **Notification metadata:** publish/widget notifications support optional `revision`, `canvasId`, and `uri`; recompute updates propagate monotonic node revision from `NodeResultRegistry`.
- **Canvas metadata propagation:** even when `revision`/`uri` are unavailable (e.g. cancellation paths), LSP should include `canvasId` when known to keep client-side stream routing deterministic.
- **Native vector stream metadata parity:** native stream consumer notifications include `revision`, `canvasId`, and `uri` when provided by call sites; this keeps `publishResult` and `publishNodeResult` routing metadata aligned with scalar/wasm paths.
- **Native fanout streaming:** when one source feeds multiple downstream inputs, native LSP fans out vector outputs in bounded chunks rather than full eager materialization.
- **LSP lifecycle cleanup:** `textDocument/didClose` is a destructive lifecycle event in `snaq-lite-lsp`: remove the closed document from `LspState`, remove graph edges where URI is source/target, remove node-result cache entry, cancel matching document/widget subscriptions with `Cancelled` notifications (`reason: "Document closed"`), then recompute downstream descendants.
- **LSP namespace isolation (soft canvas model):** custom request `snaqlite/graph/resetNamespace` accepts `{ uriPrefix }` and clears all state keyed by URI prefix (documents, edges, node-result cache, subscriptions/widgets) with cancellation notifications (`reason: "Namespace reset"`), then recomputes remaining downstream nodes that depended on removed sources.
- **URI prefix safety:** prefix-scoped removals in LSP use boundary-aware matching (`starts_with(prefix)` + boundary check unless prefix ends with `/`) to avoid accidental matches like `snaq://canvasA` also clearing `snaq://canvasA2`.
- **LSP edge compatibility after edits:** after `didOpen`/`didChange`, graph reconciliation is followed by type revalidation for affected targets (`changed_uri` and direct downstream targets). Existing edges with same input name but now-incompatible input/output types are disconnected before recompute, preventing stale wiring after signature/type edits.
- **Vector map laziness policy:** both `.map(<single-arg built-in>)` and `.map(<user fn>)` are lazy (`LazyVector::Map`), with per-element execution at stream-consume time; user functions are evaluated through `program()`/`value()` on the same database so tracked node creation remains query-safe.
- **Streaming vector access paths:** vector `length` and index access consume streams directly (`count_vector_stream`, `vector_index_stream`) instead of full collection, preserving early-stop behavior for indexing.
- **Row×column stream reductions:** row×column compare/add/sub/mul now use stream helpers (`sum_vector_stream`, `dot_product_stream`) to avoid full pair materialization.
- **LSP recompute-wave cache:** `recompute_and_push` uses a shared per-wave cache passed into graph runs to reuse node outputs and reduce redundant ancestor execution across impacted sinks.
- **LSP fetchResultSlice windowing:** for replayable lazy vectors (non-`FromInput`/non-`FromEvaluated`), `fetchResultSlice` computes `totalCount` + page window in one streaming pass without full materialization.
- **Recompute roots must include pruned targets:** when type revalidation disconnects incompatible edges, downstream targets can fall out of graph reachability from `changed_uri`; `didOpen`/`didChange` therefore recompute the union of changed URI plus target URIs where edge pruning occurred, ensuring widgets/subscriptions do not retain stale completed values.
- **Type revalidation stability:** source-side revalidation skips pruning when source parse/run is transiently invalid (no inferred output type), preserving existing edges until a concrete type is available again.
- **LSP fetchResultSlice streaming strategy:** `fetchResultSlice` computes `totalCount` + page window via streaming window extraction for replayable lazy vectors and root `FromInput` vectors, avoiding eager full-root materialization in slice paths.
- **Lock scope in fetchResultSlice:** clone `DatabaseImpl` then release `LspState` lock before heavy vector collection to reduce contention with other LSP handlers during large slice requests.

From `apps/frontend`, `pnpm run check` runs the same four steps for the frontend package only. Single command, no separate sandbox. `pnpm smoketest:dev` = run Playwright only (requires dev server already running; set `PLAYWRIGHT_REUSE_SERVER=1`). - **Frontend LSP bootstrap contract:** Worker readiness is not sufficient; browser client must send LSP `initialize` request followed by `initialized` notification before custom `snaqlite/*` RPCs, otherwise server replies `"Server not initialized"`.
- **Frontend worker loading pattern:** prefer Vite worker module import (`import WorkerCtor from './lsp.worker?worker'`) over ad-hoc URL construction for deterministic preview/build behavior.
- **Frontend optimistic state mutation pattern:** for optimistic UI updates on shared arrays/objects, always use functional state updates (`setState((current) => ...)`) and scoped rollback transforms that only undo the specific optimistic change, rather than restoring a whole captured snapshot.
- **Frontend immediate-state dependency pattern:** when a flow needs freshly returned RPC values in the same async chain (before React re-render), return and thread those values explicitly through function args rather than reading just-updated component state.
- **Chained comparisons and |x|:** **Chained:** `a < b < c` (or `<=`, `>`, `>=` in same direction) parses as `And(Lt(a,b), Lt(b,c))` (or Le/Gt/Ge). Grammar: exactly three operands (e.g. `a < b < c`); four operands parse as `(a < b < c) < d`. **Absolute value:** `| expr |` desugars to `abs(expr)` at parse time (Factor `"|" Expr "|"` → `Call("abs", [e])`). **Parser limitation (LALRPOP #596):** Expr uses 3-tier precedence (level 0: if/chained, 1: single comparison, 2: bare AddExpr). In contexts that need a full expression (e.g. RHS of `==`, `if cond then ... else ...`, vector list `[a, b]`), the generated parser only accepts the current tier, so e.g. `1 == 2` or `if 1 then 2 else 3` can fail with "expected one of ... before end of input". Workaround (ExprReset) causes shift/reduce conflicts that LALRPOP 0.21 does not resolve. Chained productions are inlined in Expr so their precedence applies.
- **Parsing:** **Program/Blocks:** Top-level is a **Program** = expression list (newline or `;` separated; blank lines allowed) wrapped in `ExprDef::Block`. Lexer: `LBrace` `{`, `RBrace` `}`, `Semicolon` `;`, `Newline` (emit on `\n` or `\r\n`); `skip_whitespace` skips only space and tab. Grammar: `Program` = `ExprList` => `Block(list)`; `ExprList` empty or `Expr ExprListRest`; `ExprListRest` can consume separator-only (blank line). **Factor** includes `"{" ExprList "}"` => `Block(list)`. Parser entry: `ProgramParser` (parse returns one `ExprDef`, the top-level Block). Custom lexer in `lexer.rs` emits `Num(NumLiteral)` (raw string + value for implicit variance), `FuncIdent(name)` when identifier is followed by `(`, else `Ident(name)`; disambiguates `sin(1)` (call) vs `sin` (symbol). Keywords: "per" (division), "as" (unit conversion; boundary check so "aspect" stays Ident), **"if", "then", "else"** (conditionals). Tokens: `~` (Tilde, explicit precision). Comparison tokens: `==`, `!=`, `<`, `<=`, `>`, `>=` (two-char before single `<`/`>`). LALRPOP grammar in `expr.lalrpop` uses extern tokens (Num, Ident, FuncIdent, etc.). Precedence: top-level Expr (comparison) ← AddExpr (+, -, as) ← Term (*, /, per, implicit mul) ← **TildeBound** (~) ← PostfixFactor (transpose) ← Factor (unary minus, NumIdents, literal, `FuncIdent "(" ArgList ")"`, Ident, parentheses). **UnitTerm:** unit-only expression (IdentIdents, *, /, per) for RHS of "as" (e.g. "m", "meters per second"). **NumIdents:** number followed by one or more idents is one factor (e.g. "1 s", "2 pi rad"); parses as left-associated Mul chain so " / 1 s" divides by 1 s and "2 pi rad" has pi→symbol, rad→unit at resolve. **IdentIdents:** one or more idents with no number (e.g. "pi", "pi rad") is one factor; "pi rad" parses as Mul(pi, rad), resolve attributes to symbol and unit. **PiIdents:** "π" alone or "π" + one or more idents (e.g. "π rad") so unicode π behaves like "pi rad". Implicit mul RHS is only number or parenthesized expr (not ident). Call args: `CallArg` = `Ident ":" Expr` (named) or `Expr` (positional); `ArgList` left-recursive.
- **Variable bindings:** **Immutable** and **lexically scoped**. Syntax: `Ident = Expr` in block or top-level (single `=`, not `==`). Scope is an **interned** Salsa type wrapping a **persistent map** (`im::OrdMap`); extension is non-destructive so (scope, expr) is the memoization key and changing a root variable produces a new scope and invalidates only dependents. **StoredValue** (Numeric, FuzzyBool, Undefined; no Vector/Symbolic in v1) is Eq+Hash for the map. **Evaluation:** `value(db, scope, expr)`; LitSymbol → **scope first**, then unit registry (1 in that unit), then symbolic (so variables **shadow** unit names, e.g. DEF=3; DEF+2 → 5). Block → iterate items, Binding extends scope, last expression is result; block of only bindings → last binding’s RHS. Resolve: bare identifiers (LitUnit) → LitSymbol (no unit lookup at resolve). Resolve and CAS (interner, convert, canonical, rewrite, substitute) handle `ExprDef::Binding` / `ExprNode::Binding` (pass-through). Lexer: `Assign` token (single `=`). Grammar: `BlockItem` = `Ident "=" BlockItem` | `Expr` (chained assignment); `ExprList` uses `BlockItem`.
- **IR:** `ExprDef` has parse-time variants `LitScalar(NumLiteral)`, `LitWithUnit(NumLiteral, String)`, `LitUnit`, `Neg(Box<ExprDef>)`, `VecLiteral(Vec<ExprDef>)`, `Transpose(Box<ExprDef>)` and resolved `Lit(Quantity)`, `LitFuzzyBool(FuzzyBool)` (constant-folded comparison), `LitSymbol(String)`, `Call(String, Vec<CallArg>)`, `As(Box<ExprDef>, Box<ExprDef>)` (unit conversion: left value expr, right unit-only expr), **`If(Box<ExprDef>, Box<ExprDef>, Box<ExprDef>)`** (condition, then_branch, else_branch), **`WithPrecision(Box<ExprDef>, Box<ExprDef>)`** (left ~ right: explicit variance from RHS value), **`Block(Vec<ExprDef>)`** (sequence of expressions; value = last or undefined if empty), **`Binding(String, Box<ExprDef>)`** (variable binding in block context). Binary `Add`, `Sub`, `Mul`, `Div`. Comparison: `Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two `Box<ExprDef>`); **result is Value::FuzzyBool**. `CallArg` = `Positional(Box<ExprDef>)` or `Named(String, Box<ExprDef>)`. Two-step flow: parse → resolve (with unit registry) → `ExprDef` with `Lit(Quantity)`, `LitFuzzyBool`, `LitSymbol`, `Call`, `As`, `VecLiteral`, `If`, or `WithPrecision` for literals. RHS of `As` resolved via `resolve_unit_expr` (only LitUnit/Mul/Div → Lit(Quantity), no symbols). `ExprData` has `LitFuzzyBool(FuzzyBool)`, `Call(name, Vec<(Option<String>, Expression)>)`, `As(Expression, Expression)`, `VecLiteral(Vec<Expression>)`, `Transpose(Expression)`, **`WithPrecision(Expression, Expression)`**, and **`If(Expression, Expression, Expression)`**, **`Block(Vec<Expression>)`**, **`Binding(String, Expression)`**; `value(db, scope, expr)` returns `Result<Value, RunError>`; scope is interned, empty via `empty_scope(db)`. Block: evaluate expressions in order; return last value or `Value::Undefined` if empty.
- **Numbers and quantities:** Numeric values are `SnaqNumber` (value + variance). Variance propagates through arithmetic (Add/Sub: sum of variances; Mul/Div: standard propagation; scale by f64: variance × factor²). **Literal variance:** Lexer emits `NumLiteral { raw, value }`; `LitScalar`/`LitWithUnit` carry `NumLiteral`. Resolve uses `implicit_variance()`: no decimal point → abs_err 0.5, variance 0.25; N decimal places (mantissa only) → abs_err = 5×10^−(N+1), variance = abs_err². **Explicit precision:** `expr ~ expr` (tilde) → `WithPrecision(l, r)`; precedence above * / + −. Evaluation: result = left value, variance = (right.value())²; right's variance discarded; RHS must be > 0 → `RunError::PrecisionMustBePositive`. `Quantity` holds `number: SnaqNumber` and unit; `with_number(SnaqNumber, unit)` for explicit variance.
- **Units:** Default registry (`default_si_registry`) provides all 7 SI base units (m, kg, s, A, K, mol, cd) and all SI derived units with special names (J, C, V, F, ohm, S, Wb, T, H, Hz, N, Pa, W, lm, lx, Bq, Gy, Sv, kat), plus km, g, hour, minute, second, day, week, month, quarter, year, decade, century, millennium, mile, au, parsec, light_year, nautical_mile (nmi), joule, eV, celsius, fahrenheit (scale 5/9 for differences; absolute zero not modeled), Angle (rad, degree, arcmin, arcsec), **Area** (m2, are, hectare; prefix + m2 → (prefix·m)²), **Volume** (m3, liter, milliliter; prefix + m3 → (prefix·m)³ via CUBED_LENGTH_REST), **Concentration** (molar, M), **Velocity** (knot), pressure (bar, atm, psi, mmHg, torr), mass (pound, ounce), energy (calorie, BTU, Wh, kWh), power (horsepower, hp), dimensionless (percent, ppm, ppb), solid angle (steradian, sr). Squared-length: prefix + "m2" or "m²" in `try_lookup` resolves to (prefix·m)²; cubed-length: CUBED_LENGTH_REST for km3, cm3, etc. Registry stores singular forms; plural input normalized via `singularize`. Long-form aliases via `add_derived_alias`. Prefix resolution via `get_unit_with_prefix`. Quantity has `full_simplify` and `full_simplify_with_registry`. **Symbol registry** default also has physical constants: c, h, hbar, R (dimensionless numeric values; use with units e.g. c * m / s).
- **Errors:** Runtime errors can carry an optional **span** (RunError::at, RunError.span). **Dimension labels in messages:** Dimension mismatch and incompatible-units messages show each unit via `Unit::dimension_label_for_error()` (scalar → "none", else unit display). When present and source is passed to format_run_error_with_source, output includes "at line X, column Y" and a snippet with squiggle (same style as parse errors). **Span propagation:** run()/run_with_registry() set ProgramDef::spanned_root so the expression graph gets spans; value() uses run_err_with_span(db, expr, kind) for direct errors and with_span_if_missing for errors from helpers; add_values, sub_values, mul_values, div_values, cmp_values, neg_value accept Option<Span> and use RunError::at when Some; eval_call/eval_if errors get call-site span via with_span_if_missing; map result collection and vector method branches (sum, mean, min, max, product, variance, stddev) attach the MethodCall span to helper errors via with_span_if_missing. CAS rewrite attaches the failing node's span to quantity_to_run_error and comparison errors. resolve_unit_expr and substitute_symbols_spanned_inner use RunError::at(span, ...) for UnknownUnit and SymbolHasNoValue. LALRPOP ParseError is mapped to crate::error::ParseError in the parser wrapper. Generated code has targeted `#[allow(...)]` for Clippy. Transpose on non-vector → `RunError::ExpectedVector`. Vector op length mismatch (element-wise or dot) → `RunError::VectorLengthMismatch { left_len, right_len }`. If condition not FuzzyBool → `RunError::ExpectedCondition`. If branches not blendable (boolean/vector) → `RunError::IfBranchTypeMismatch`. Tilde RHS ≤ 0 or non-finite → `RunError::PrecisionMustBePositive`. to_quantity on Undefined → `RunError::UndefinedResult`. Vector reduction (mean, min, max, variance, stddev) on empty → `RunError::EmptyVectorReduction(method)`. Built-in invalid argument (e.g. sqrt of negative) → `RunError::InvalidArgument(msg)`.
- **±∞:** Only +∞ and -∞ supported (no NaN). Division: nonzero/0 → ±∞; 0/0 → `Value::Undefined` (not an error at Value level; run_numeric("0/0") → UndefinedResult). Infinite values have variance 0; display and arithmetic via f64. `Quantity::is_infinite()` / `is_finite()`; `convert_to` short-circuits when value is infinite (unit change only).
- **Value and symbols:** `Value` is `Numeric(Quantity)`, `Symbolic(SymbolicQuantity)`, **FuzzyBool(FuzzyBool)** (comparison result: True, False, or Uncertain(prob)), **Undefined** (empty block/program), `Vector(VectorValue)`, **`Function(UserFunction)`**, or **`BuiltinFunction(String)`** (built-in by name, e.g. sqrt; usable where a lambda is required). LitSymbol resolution: **scope first**, then **built-in function name** (if `functions::param_names(name).is_some()` → `Value::BuiltinFunction(name)`), then unit registry, then symbolic. StoredValue includes `BuiltinFunction(String)` so e.g. `f = sqrt` works. Display: Numeric/Symbolic as before; FuzzyBool as "true"/"false"/"uncertain(prob)". `to_quantity` on FuzzyBool or on Symbolic with comparison expr → `RunError::BooleanResult`. Comparisons use statistical P(A>B) and CONFIDENCE_THRESHOLD (0.95); when variance is zero or means equal (for Ge/Le) result is crisp. `SymbolRegistry` maps symbol names to numeric values (built-ins: pi, π, e, sqrt_2, sqrt_3, phi, c, h, hbar, R). Bare identifiers resolve to `LitSymbol`; at eval time LitSymbol is resolved **scope first**, then unit registry, then symbolic. **run_numeric** uses `substitute_symbols(def, symbol_registry, unit_registry)`, which collects names bound in the program and leaves them as LitSymbol so `value()` resolves them from scope; other identifiers are substituted from symbol registry then unit registry. So variable bindings that shadow unit names affect both `run()` and `run_numeric()`. Evaluation returns `Value`. `SymbolicExpr` (Number, Symbol, Sum, Product, Neg, Div, Call, Eq, Ne, Lt, Le, Gt, Ge); `SymbolicQuantity(expr, unit)`. Comparison expressions: numeric comparison → `Value::FuzzyBool`; symbolic comparison stays Symbolic; `expr.is_comparison()` used to reject `to_quantity`. Default: symbolic result; `run_numeric()` on comparison returns `BooleanResult`. CLI: default symbolic, `--numeric` for numeric. Error `SymbolHasNoValue(name)` when substituting and symbol has no value.
- **Functions:** Built-ins in `functions.rs`: trig (sin, cos, tan); inverse trig (asin, acos, atan, atan2); hyperbolic (sinh, cosh, tanh, asinh, acosh, atanh); elementary (sqrt, cbrt, abs, sign, floor, ceil, round, trunc, mod); exp/log (exp, ln, log, log10, log2); pow, cbrt; probability (norm_cdf, norm_inv, factorial, binom, perm); gcd, lcm; max, min; take. Symbolic sugar: `^` → pow, `%` → mod, `!` postfix → factorial, `√` prefix → sqrt, `∛` prefix → cbrt. Statistics: corr(X,Y) (eval_call special-case), V.median(), V.quantile(p). For well-known angles (0, π/6, π/4, π/3, π/2, π and degree 0°, 30°, 45°, 60°, 90°, 180°), trig returns symbolic exact results (e.g. √2/2, √3/2 via constants sqrt_2, sqrt_3). `KnownAngle`, `known_angle_from_rad`, `known_angle_from_symbolic`, `exact_sin/cos/tan`; eval_call uses them so symbolic run keeps constants (π, √2, √3) in display and run_numeric yields a number. Each built-in declares param names; call-site args bound positionally then by name. Unknown function → `RunError::UnknownFunction`. Wrong dimension for trig → `RunError::ExpectedAngle { actual }`. Symbolic calls: if any arg is symbolic and not a known angle, result is `Value::Symbolic(SymbolicExpr::Call(name, args))`. Variance is propagated through trig via first-order propagation.
- **User-defined functions:** Syntax `fn (params) => (expr)` or `=> { block }` (anonymous), `fn name(params) => ...` (named; FnName = Ident | FuncIdent so `fn f(a,b) => ...` parses). Params: `Ident` or `Ident = Expr` (default). IR: `ExprDef::Lambda`, `ExprDef::CallExpr`; `ExprData` same. Value: `Value::Function(UserFunctionId)`; scope: `StoredValue::Function(id)`. `user_function.rs`: `UserFunction` (params, body, closure_env), thread-local registry (register/get). **Call:** eval_call resolves name in scope first; if `StoredValue::Function(id)` → eval_user_call (bind positional, then named args, fill missing from defaults in closure scope, extend closure env, evaluate body); else built-in path. **Errors:** missing required arg, unknown parameter name (named arg not in params), duplicate parameter, too many positional args, not callable. **Cannot shadow built-ins:** eval_binding_chain checks `functions::param_names(name).is_some()` before extend → `RunError::CannotObfuscateBuiltin(name)`. CAS: Lambda/CallExpr in interner, convert, canonical, rewrite, substitute (recurse; substitute treats lambda param names as bound). Display: `Value::Function` → `"<function>"`; when program result is a function, run_format and CLI show `<function>` (run_numeric / to_quantity on function → BindingValueNotSupported). **CLI `-n`:** single evaluation via `run()` then branch (Function → print `<function>`, else `to_quantity` and format).
- **Conditionals (if/then/else):** Syntax `if condition then expression else expression`; condition and branches are full expressions. Condition must evaluate to **FuzzyBool**. **Crisp:** True → only then branch evaluated; False → only else branch. **Superposition (Uncertain(p)):** both branches evaluated and blended. Numeric: both quantities converted to common unit (`Quantity::smaller_unit`), then blended mean = p·mean_A + (1−p)·mean_B and variance via Law of Total Variance (inner + between). Symbolic: weighted sum P·then_expr + (1−P)·else_expr as SymbolicExpr (same dimension required). Type mismatch: both branches must be numeric or symbolic; FuzzyBool or Vector → `RunError::IfBranchTypeMismatch`. Condition not FuzzyBool → `RunError::ExpectedCondition`. CAS: `ExprNode::If(cond, then, else)`; rewrite constant-folds when condition is `LitFuzzyBool(True)` or `LitFuzzyBool(False)` to single branch.
- **CAS:** `cas/` module: interned AST (`ExprId`, `ExprNode`, `ExprInterner`), canonicalization (flatten Add/Mul, sort by rank), rewrite (identity/annihilation, constant fold, like-term collection). **LitFuzzyBool:** `ExprNode::LitFuzzyBool(FuzzyBool)`; rewrite constant-folds comparisons using statistical P(A>B) and CONFIDENCE_THRESHOLD to `LitFuzzyBool(True/False/Uncertain(prob))`. **If:** `ExprNode::If(ExprId, ExprId, ExprId)`; rewrite folds to then/else branch when condition is LitFuzzyBool(True/False). **WithPrecision:** `ExprNode::WithPrecision(ExprId, ExprId)`; pass-through in canonicalization and rewrite. **Block:** `ExprNode::Block(Vec<ExprId>)`; order preserved in canonicalize and rewrite (recurse into each child). **Comparison nodes:** `ExprNode::Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two ExprId); pass-through in canonicalization; rewrite when both children Lit → `LitFuzzyBool`. **Mul operands:** sorted by rank except when both operands are vector-valued (`VecLiteral` or `Transpose`): then order is preserved so row×column (dot) vs column×row (outer) is not swapped. `ExprNode::VecLiteral(Vec<ExprId>)` pass-through. `simplify_symbolic(def, registry)` and `simplify_numeric(def, unit_registry, symbol_registry)`; `substitute_symbols(def, symbol_registry, unit_registry)` substitutes LitSymbol from symbol registry (scalar) or unit registry (1 in that unit), then CAS; handles `LitFuzzyBool` and `WithPrecision`. Run pipelines apply CAS before building Salsa program. Division in rewrite: nonzero/0 constant-folds to ±∞; 0/0 left as Div so value() maps it to `Value::Undefined`.
- **Dates (temporal literals):** Syntax `@YYYY`, `@YYYY-MM`, `@YYYY-MM-DD`, `@YYYY-MM-DDTHH`, `@YYYY-MM-DDTHH:MM`, `@YYYY-MM-DDTHH:MM:SS` (ISO 8601, UTC). Lexer: `Tok::TemporalLiteral(String)`, `take_temporal_literal`; invalid/incomplete → `LexicalError::InvalidTemporal`. IR: `LitTemporal(String)` at parse; resolve → `LitDate(GranularDate)` via `date::parse_temporal_literal` (grain inference, anchor in Unix seconds). `GranularDate { anchor_epoch_sec, grain }` in `date.rs`; `min_epoch_sec`/`max_epoch_sec` for interval bounds. Value/Scope: `Value::Date(GranularDate)`, `StoredValue::Date`; CAS LitDate pass-through. Algebra: Date ± Time (duration must have Time dimension; duration grain must not be finer than date grain → `IncompatibleDateGrain`); Date − Date → seconds (Quantity). Comparison: Date vs Date → interval overlap → `FuzzyBool` (true/false/uncertain(0.5)); Date vs non-Date → `InvalidArgument`. Single-comparison RHS in grammar is `AddExpr` (not `Expr`) so e.g. `@2025 < @2026` parses. Docs: docs/DATES.md.
- **Maps:** Map literals `{ key: value, ... }` (at least one entry; keys unquoted idents). Empty `{}` is a block. **Access:** `.key` (Member) and `[key]` (Index); bracket content is the key string (trimmed). Missing key → `Value::Undefined`. **Storage:** `map_registry` is **shared** (Arc<Mutex<RegistryStore>>) so stream feeders (other threads) can register row-as-map and main thread can get_key; `Value::Map(u64)`, `StoredValue::Map(MapId)`; scope from_value/to_value. **Record/stream:** `Record` = Vec<(String, Value)>; `record_to_value(entries)` and `record_to_chunk_element(entries)` for hosts building tabular stream chunks. IR: `MapLiteral(Vec<(String, Box<ExprDef>)>)`; CAS (interner, convert, canonical, rewrite, substitute) recurse on value exprs. Member/Index on Value::Map use map_registry::get_key. Display: `<map>`; to_quantity on map → error. Bracket key: lexer emits BracketKey(trimmed) when `[` follows postfix factor; vector index interprets key as non-negative int or variable.
- **External stream inputs ($name):** Programs can refer to external data via `$name`. If the handle is missing at **consume** time (vector_into_stream), the stream yields one **StreamInputNotAvailable** error then completes (OneErrorStream). Lexer: `DollarIdent(name)`; grammar Factor → `ExternalStream(name)`; IR `ExprDef`/`ExprData`/`SpannedExprDefKind::ExternalStream`; resolve and CAS pass-through. Salsa input **StreamInputRegistry** (name → `StreamHandleId`); thread-local **stream handle registry** maps id → receiver (`futures::channel::mpsc::UnboundedReceiver<Chunk>`). **Chunk** = `Vec<Result<Option<Value>, RunError>>`; Host pushes chunks, drops sender for EOF. **create_stream_input()** in stream_handle returns (StreamHandleId, UnboundedSender<Chunk>) so Host can create and retain sender. `value()` for ExternalStream: read registry, return `Value::Vector(LazyVector::FromInput(handle_id))` or `UnboundStreamInput`. **LazyVector::FromInput(StreamHandleId)**; `vector_into_stream` takes receiver via `take_receiver(id)`, wraps in **ChunkFlattenStream** (flattens chunks to elements). Host: create channel (or create_stream_input), `register(receiver)` → id, set `StreamInputRegistry` with name→id, `run_with_stream_inputs` → (Value, db), then `vector_into_stream` on vector result while pushing chunks. Docs: docs/EXTERNAL_STREAMS.md. **CLI:** `--stream name=path`; **--stream-variance zero|infer** (default zero): zero = exact values (variance 0); infer = variance from decimal places in text (CSV cells, newline-delimited lines; same rule as NumLiteral). Parquet/Arrow always use zero (no decimal representation). Format detected by extension (`.csv` → tabular via snaq-lite-ingest; else newline-delimited numbers); stream_feed_dispatch::feed_stream_file_to_sender(path, sender, variance_mode); duplicate name → error; feeder thread join checked (panic → exit 1); numeric output via shared `print_numeric_quantity`. **Stream variance (lang):** `StreamVarianceMode`, `decimal_string_to_quantity(s)` in stream_variance.rs. **Tabular (snaq-lite-ingest):** TabularParser trait (object-safe: parse(Box<dyn ReadSeek>), ReadSeek = Read + Seek + Send); StreamFormat enum (Csv, Parquet, Arrow, NumericLines), detect_format(path, magic), parse_tabular(); CSV, Parquet, Arrow IPC (optional `parquet` feature). Parsing streams row-by-row; Parquet/Arrow read input into memory (format requires footer or try-both), then yield Record per row. record_batch::row_to_record converts Arrow RecordBatch row to Record; numeric types → Value::Numeric, null/other → Undefined. New formats: implement TabularParser, extend StreamFormat and detect_format.
- **WASM Host (browser):** snaq-lite-wasm exposes JS API: **createStreamInput()** → stream index; **runWithStreamInputs(script, streamInputMap)** → { runIndex, isVector }; **pushChunk(streamIndex, array)** (array of numbers or null); **closeStream(streamIndex)**; **startFeeder(streamIndex, ReadableStream)** (reads bytes, parses newline-delimited numbers, pushes chunks, yields to event loop); **consumeOutputStream(runIndex, onChunk, onDone, onError)** (spawn_local drives vector_into_stream, yields every 16 elements). **LSP WASM** (stream_host): same plus **startCsvFeeder(streamIndex, ReadableStream)** (first line = headers, rest = CSV rows → row-as-map chunks; comma/semicolon delimiter); **pushChunkMaps(streamIndex, rowsJs)** for JS array of row objects. Worker messages **feedStreamFromUrl** / **feedStreamFromReadableStream** accept optional **format** (`numeric` | `csv`) and call startFeeder or startCsvFeeder accordingly. Host module (host.rs): thread-local STREAM_HANDLES (index → id + sender), RUN_RESULTS (run index → Option<(Value, Database)>); value_to_js serializes Value for callbacks. No file paths; 4GB memory safeguard; event-loop yielding. See docs/EXTERNAL_STREAMS.md "WASM Host (browser)".
- **LSP (snaq-lite-lsp):** Dual-target: **native** (Tokio stdio, `run_native`, binary `snaq-lite-lsp`) and **WASM** (Web Worker, `transport.rs`: WasmMessageReader/WasmMessageWriter, `start_snaq_lite_lsp(postMessageCallback)`, `push_lsp_message(s)`). Shared backend `SnaqLiteBackend` uses async locks for `LspState`, subscriptions, graph state, widget registry, and **node result registry** (URI -> latest value/error revision). Text sync is **incremental** (`TextDocumentSyncKind::INCREMENTAL`) with full-text replacement fallback when change has no range. Position handling is centralized in `src/position.rs` and uses UTF-16 LSP positions for byte-span conversions. **Graph runtime model:** on didOpen/didChange/connect/disconnect, reconcile target input schema (`reconcile_target_inputs`), compute transitive impacted closure (`impacted_from_changed_nodes`), eagerly recompute impacted nodes, update node result store, and push **Completed/Error updates** to existing doc/widget subscriptions (do not cancel on ordinary edits). Custom methods: `snaqlite/subscribe`, `snaqlite/unsubscribe`, `snaqlite/graph/connect`, `snaqlite/graph/disconnect`, `snaqlite/graph/subscribeWidget`, `snaqlite/graph/unsubscribeWidget`, `snaqlite/graph/fetchResultSlice`; notifications: `snaqlite/publishResult`, `snaqlite/graph/nodeSignatureUpdated`, `snaqlite/graph/widgetDataUpdate`. See docs/LSP.md.
- **Result handle canonicalization:** LSP completed payloads may include `resultHandle`; `fetchResultSlice` accepts either legacy `widgetId` or canonical `{ resultHandle, cursor }`, returning optional `nextCursor` for one-time continuation paging.
- **Result handle lifecycle registry:** `ResultHandleRegistry` owns URI-scoped result handles and cursor tokens; recompute/upsert refreshes handle revision+value and invalidates old cursors; close/reset/import/shutdown remove matching handles/cursors.
- **Deferred connect validation:** `graph/connect` allows edges when source output type is temporarily unknown; once source type resolves, existing revalidation flow keeps compatible edges and prunes incompatible ones.
- **Canvas rebind gate:** hard single-canvas isolation still applies, but rebind is allowed only when runtime is fully drained (no open docs, subscriptions, widgets, or result handles/cursors).
- **Canvas rebind in document lifecycle:** `didOpen`/`didChange` must use the same rebind-aware guard (`ensure_canvas_uri_request`) as graph RPCs; direct `state.ensure_canvas_uri` in these handlers bypasses rebind-after-drain and can silently leave rebound documents unopened.
- **Graph param helper RPCs:** optional source-authoritative helpers (`renameParam`, `addParam`, `removeParam`) mutate input declarations in document text and then run the same reconcile/revalidate/recompute pipeline as `didChange`.
- **Graph param rename canonicalization:** `renameParam` must trim `newName` before writing updated `input name@paramId: Type` source to avoid persisting whitespace-padded identifiers.
- **Graph add-param normalization:** `addParam` must normalize (`trim`) incoming `name`, `paramId`, and `typeName` once, then use normalized values for non-empty checks, duplicate detection, and source generation; mixing raw + trimmed fields allows whitespace-padded duplicates to slip through.
- **LSP reconcile safety on syntax errors:** Document entries track `parse_succeeded`. `reconcile_graph_for_uri` only prunes edges when parse succeeded for the current document version, preventing temporary syntax errors during editing from irreversibly removing graph connections.
- **Vectors:** `Value::Vector(VectorValue)` where `VectorValue { inner: LazyVector, orientation: VectorOrientation }`. Orientation: **Column** (default) or **Row** (after transpose). Lazy vector forms include `FromEvaluated`, `Map`, `ZipMap`, `Outer`, `Transpose`, `Take`, and `FromInput`. **Map laziness:** built-in and user-function mapping use `LazyVector::Map` and apply per element during streaming. **Streaming access/reductions:** `V.length`, index access, `sum/mean/min/max/product/variance/stddev/all/any`, and row×column reductions are stream-driven (no eager full collect in the method path). **Multi-dimensional ops:** transpose and outer product stay API-lazy but buffer required row/left data as mathematically necessary.

- **Semantic-change-gated LSP recompute:** graph/text edits (`didOpen`/`didChange`) reevaluate changed roots but only propagate to downstream nodes when upstream value/error changes. Topology mutations (connect/disconnect/close/reset/import and edge-pruning revalidation) still force downstream reevaluation.
- **Input declaration binding policy:** vector-typed declared inputs are lazy `LazyVector::FromInput` on both native and WASM; native keeps scalar-friendly binding for non-vector declarations when a single value is available.
- **Non-replayable slice contract:** `fetchResultSlice` does not materialize `FromInput` root results; non-replayable random access (`offset > 0` or cursor) is rejected with `invalid_params`.
- **Forward-only non-replayable handle paging:** for handle-scoped non-replayable lineages, first page must start at `offset=0`, later pages require cursor continuation; direct `offset>0` without cursor is rejected with `invalid_params`.
- **Non-replayable handle cache upgrade:** first forward-only page read for a handle-backed `FromInput` root may upgrade that handle value to `FromEvaluated` for deterministic continuation paging, while preserving the handle lineage's forward-only offset contract.
- **Graph type compatibility unification:** connect-time and revalidation-time edge checks use the same compatibility matrix helper (`are_graph_types_compatible`) to avoid divergent wiring/pruning decisions.
- **Non-replayable nested-path contract:** `fetchResultSlice` rejects nested `path` access for `FromInput` roots when addressed by `resultHandle`, preventing implicit replay/materialization through path traversal.
- **Forward-only handle paging state:** `fetchResultSlice` for `resultHandle` + `FromInput` roots must use cursor-gated continuation state (single-consumer stream ownership in `ResultHandleRegistry`) and must not upgrade handle values to `FromEvaluated`.
- **WASM progress non-consumption policy:** `send_wasm_vector_progress_notifications` must not call `vector_into_stream` for `LazyVector::FromInput`; otherwise it consumes the receiver before `fetchResultSlice` continuation reads.
- **Forward-only pagination metadata:** for non-replayable handle paging, `totalCount` may be a lower bound until stream exhaustion; clients should use `hasMore`/`nextCursor` for continuation.
- **Vector completion transport policy:** subscription vector `Completed` payloads use summary + optional `resultHandle`; no eager full-vector `display` payload is emitted.
- **Graph URI contract:** graph RPCs (`graph/*`, widget APIs, graph connect/disconnect, fetchResultSlice by graph source URI) require canonical `snaq://<canvasId>/...` URIs; legacy document-root `subscribe`/`subscribeNode` still accept non-graph URI flows for compatibility.
- **Atomic graph patch API:** `snaqlite/graph/applyPatch` stages and validates multi-op graph/source mutations (`setNodeSource`, `connect`, `disconnect`, `renameParam`, `addParam`, `removeParam`) and commits them in one recompute wave; failures abort without partial source/edge commit.
- **Patch-source edge pruning parity:** `graph/applyPatch` must keep staged graph wiring aligned with source edits before commit. Source-mutating ops reconcile staged target inputs; `removeParam` explicitly disconnects its `paramId` edge in staged graph to prevent stale exported edges even when updated source is temporarily unparseable.
- **Bootstrap runtime snapshot API:** `snaqlite/bootstrapSession` returns lightweight runtime snapshot (`canvasId`, open document count, active subscriptions/widgets/handles, drained flag) for worker/client recovery orchestration.
- **Param helper source edits:** `renameParam`/`addParam`/`removeParam` are span-aware over parsed `InputDecl` nodes (not line-split rewriting), preserving non-input source content and enforcing identifier validation for names/paramIds.
- **Deferred vector literal evaluation:** vector literals can use deferred element evaluation with captured lexical env (`FromExprsWithEnv`) so consumers evaluate elements on pull instead of eager full literal evaluation.
- **Session runtime API:** `snaq-lite-lang::Session` is the persistent DB entrypoint for repeated evaluations; one-shot `run*` helpers remain compatibility wrappers that create ephemeral sessions.
- **Reducer stream policy:** `norm` and `corr` are online streaming reducers; exact order-statistics reducers (`median`, `quantile`) reject non-replayable streams unless materialized upstream.
- **Approximate order-stat reducers:** `median_approx` / `quantile_approx(p)` are single-pass streaming reducers (reservoir-sampled quantile approximation) and are valid on forward-only streams.
- **Native non-vector input binding stream policy:** for `input name: <non-Vector>`, native binding peeks stream cardinality (single item => scalar-friendly binding) but does not fully materialize multi-item streams; multi-item inputs are forwarded to a fresh lazy `FromInput` handle.
- **Bounded multidimensional buffering:** transpose and column×row outer-product remain API-lazy but enforce bounded buffering with fail-fast runtime errors. Limits are configured by `SNAQ_TRANSPOSE_BUFFER_LIMIT` and `SNAQ_OUTER_LEFT_BUFFER_LIMIT` (default `100000` each).
- **Forward-only display safety:** formatting paths must never consume `LazyVector::FromInput`; runtime display helpers return a non-draining summary token (`<stream-vector>`) for forward-only vectors.
- **Subscription publish dedup pattern:** `subscribe` and `subscribeNode` should route scalar/vector completion notifications through shared internal helpers to keep metadata parity and reduce drift.
- **Node semantic fingerprint strategy:** LSP node-result change detection uses explicit semantic hashing (value-shape-aware, handle-aware for `FromInput`) instead of heavy full debug-string fingerprints, while preserving revision semantics.
- **Deferred-vector fingerprint completeness:** `LazyVector::FromExprsWithEnv` fingerprints must include both `defs` and captured `Env`; omitting `Env` can suppress revision bumps when lexical bindings change.
- **Function-value fingerprint completeness:** `Value::Function` fingerprints must include full `UserFunction` payload (params, body, closure env id), not just variant tag, so function-node updates propagate through semantic-change gating.
- **Bridge scenario non-blocking UX pattern:** UI orchestration flows should not block end-to-end completion on optional paged slice reads; manual slice actions remain available independently from scenario completion.
