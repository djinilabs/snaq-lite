# System patterns

- **Root scripts:** Root `package.json` is script-only (no deps). `pnpm run install:frontend` installs frontend deps from root. `pnpm test` (backend then frontend), `pnpm build` (both), `pnpm run lint` (backend clippy + frontend ESLint; both must pass), `pnpm dev` / `pnpm preview` (frontend). Backend = Rust workspace (cargo); frontend = `apps/frontend` (pnpm -C apps/frontend).
- **Frontend (apps/frontend):** Use **pnpm** Ōēź10.13.1 (packageManager in package.json; pnpm-lock.yaml committed). **ESLint:** flat config (eslint.config.js), @eslint/js + typescript-eslint + react + react-hooks, recommended rules; `pnpm run lint` runs `eslint . --max-warnings 0` (no warnings allowed). Lint-rule tests in src/eslint/lint-rules.test.ts (programmatic ESLint with test-eslint.config.js). TanStack Start SPA (no SSR). Build LSP WASM: pnpm run build:lsp-wasm (output apps/frontend/public/lsp-wasm/). - **Chained comparisons and |x|:** **Chained:** `a < b < c` (or `<=`, `>`, `>=` in same direction) parses as `And(Lt(a,b), Lt(b,c))` (or Le/Gt/Ge). Grammar: exactly three operands (e.g. `a < b < c`); four operands parse as `(a < b < c) < d`. **Absolute value:** `| expr |` desugars to `abs(expr)` at parse time (Factor `"|" Expr "|"` ŌåÆ `Call("abs", [e])`). **Parser limitation (LALRPOP #596):** Expr uses 3-tier precedence (level 0: if/chained, 1: single comparison, 2: bare AddExpr). In contexts that need a full expression (e.g. RHS of `==`, `if cond then ... else ...`, vector list `[a, b]`), the generated parser only accepts the current tier, so e.g. `1 == 2` or `if 1 then 2 else 3` can fail with "expected one of ... before end of input". Workaround (ExprReset) causes shift/reduce conflicts that LALRPOP 0.21 does not resolve. Chained productions are inlined in Expr so their precedence applies.
- **Parsing:** **Program/Blocks:** Top-level is a **Program** = expression list (newline or `;` separated; blank lines allowed) wrapped in `ExprDef::Block`. Lexer: `LBrace` `{`, `RBrace` `}`, `Semicolon` `;`, `Newline` (emit on `\n` or `\r\n`); `skip_whitespace` skips only space and tab. Grammar: `Program` = `ExprList` => `Block(list)`; `ExprList` empty or `Expr ExprListRest`; `ExprListRest` can consume separator-only (blank line). **Factor** includes `"{" ExprList "}"` => `Block(list)`. Parser entry: `ProgramParser` (parse returns one `ExprDef`, the top-level Block). Custom lexer in `lexer.rs` emits `Num(NumLiteral)` (raw string + value for implicit variance), `FuncIdent(name)` when identifier is followed by `(`, else `Ident(name)`; disambiguates `sin(1)` (call) vs `sin` (symbol). Keywords: "per" (division), "as" (unit conversion; boundary check so "aspect" stays Ident), **"if", "then", "else"** (conditionals). Tokens: `~` (Tilde, explicit precision). Comparison tokens: `==`, `!=`, `<`, `<=`, `>`, `>=` (two-char before single `<`/`>`). LALRPOP grammar in `expr.lalrpop` uses extern tokens (Num, Ident, FuncIdent, etc.). Precedence: top-level Expr (comparison) ŌåÉ AddExpr (+, -, as) ŌåÉ Term (*, /, per, implicit mul) ŌåÉ **TildeBound** (~) ŌåÉ PostfixFactor (transpose) ŌåÉ Factor (unary minus, NumIdents, literal, `FuncIdent "(" ArgList ")"`, Ident, parentheses). **UnitTerm:** unit-only expression (IdentIdents, *, /, per) for RHS of "as" (e.g. "m", "meters per second"). **NumIdents:** number followed by one or more idents is one factor (e.g. "1 s", "2 pi rad"); parses as left-associated Mul chain so " / 1 s" divides by 1 s and "2 pi rad" has piŌåÆsymbol, radŌåÆunit at resolve. **IdentIdents:** one or more idents with no number (e.g. "pi", "pi rad") is one factor; "pi rad" parses as Mul(pi, rad), resolve attributes to symbol and unit. **PiIdents:** "ŽĆ" alone or "ŽĆ" + one or more idents (e.g. "ŽĆ rad") so unicode ŽĆ behaves like "pi rad". Implicit mul RHS is only number or parenthesized expr (not ident). Call args: `CallArg` = `Ident ":" Expr` (named) or `Expr` (positional); `ArgList` left-recursive.
- **Variable bindings:** **Immutable** and **lexically scoped**. Syntax: `Ident = Expr` in block or top-level (single `=`, not `==`). Scope is an **interned** Salsa type wrapping a **persistent map** (`im::OrdMap`); extension is non-destructive so (scope, expr) is the memoization key and changing a root variable produces a new scope and invalidates only dependents. **StoredValue** (Numeric, FuzzyBool, Undefined; no Vector/Symbolic in v1) is Eq+Hash for the map. **Evaluation:** `value(db, scope, expr)`; LitSymbol ŌåÆ **scope first**, then unit registry (1 in that unit), then symbolic (so variables **shadow** unit names, e.g. DEF=3; DEF+2 ŌåÆ 5). Block ŌåÆ iterate items, Binding extends scope, last expression is result; block of only bindings ŌåÆ last bindingŌĆÖs RHS. Resolve: bare identifiers (LitUnit) ŌåÆ LitSymbol (no unit lookup at resolve). Resolve and CAS (interner, convert, canonical, rewrite, substitute) handle `ExprDef::Binding` / `ExprNode::Binding` (pass-through). Lexer: `Assign` token (single `=`). Grammar: `BlockItem` = `Ident "=" BlockItem` | `Expr` (chained assignment); `ExprList` uses `BlockItem`.
- **IR:** `ExprDef` has parse-time variants `LitScalar(NumLiteral)`, `LitWithUnit(NumLiteral, String)`, `LitUnit`, `Neg(Box<ExprDef>)`, `VecLiteral(Vec<ExprDef>)`, `Transpose(Box<ExprDef>)` and resolved `Lit(Quantity)`, `LitFuzzyBool(FuzzyBool)` (constant-folded comparison), `LitSymbol(String)`, `Call(String, Vec<CallArg>)`, `As(Box<ExprDef>, Box<ExprDef>)` (unit conversion: left value expr, right unit-only expr), **`If(Box<ExprDef>, Box<ExprDef>, Box<ExprDef>)`** (condition, then_branch, else_branch), **`WithPrecision(Box<ExprDef>, Box<ExprDef>)`** (left ~ right: explicit variance from RHS value), **`Block(Vec<ExprDef>)`** (sequence of expressions; value = last or undefined if empty), **`Binding(String, Box<ExprDef>)`** (variable binding in block context). Binary `Add`, `Sub`, `Mul`, `Div`. Comparison: `Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two `Box<ExprDef>`); **result is Value::FuzzyBool**. `CallArg` = `Positional(Box<ExprDef>)` or `Named(String, Box<ExprDef>)`. Two-step flow: parse ŌåÆ resolve (with unit registry) ŌåÆ `ExprDef` with `Lit(Quantity)`, `LitFuzzyBool`, `LitSymbol`, `Call`, `As`, `VecLiteral`, `If`, or `WithPrecision` for literals. RHS of `As` resolved via `resolve_unit_expr` (only LitUnit/Mul/Div ŌåÆ Lit(Quantity), no symbols). `ExprData` has `LitFuzzyBool(FuzzyBool)`, `Call(name, Vec<(Option<String>, Expression)>)`, `As(Expression, Expression)`, `VecLiteral(Vec<Expression>)`, `Transpose(Expression)`, **`WithPrecision(Expression, Expression)`**, and **`If(Expression, Expression, Expression)`**, **`Block(Vec<Expression>)`**, **`Binding(String, Expression)`**; `value(db, scope, expr)` returns `Result<Value, RunError>`; scope is interned, empty via `empty_scope(db)`. Block: evaluate expressions in order; return last value or `Value::Undefined` if empty.
- **Numbers and quantities:** Numeric values are `SnaqNumber` (value + variance). Variance propagates through arithmetic (Add/Sub: sum of variances; Mul/Div: standard propagation; scale by f64: variance ├Ś factor┬▓). **Literal variance:** Lexer emits `NumLiteral { raw, value }`; `LitScalar`/`LitWithUnit` carry `NumLiteral`. Resolve uses `implicit_variance()`: no decimal point ŌåÆ abs_err 0.5, variance 0.25; N decimal places (mantissa only) ŌåÆ abs_err = 5├Ś10^ŌłÆ(N+1), variance = abs_err┬▓. **Explicit precision:** `expr ~ expr` (tilde) ŌåÆ `WithPrecision(l, r)`; precedence above * / + ŌłÆ. Evaluation: result = left value, variance = (right.value())┬▓; right's variance discarded; RHS must be > 0 ŌåÆ `RunError::PrecisionMustBePositive`. `Quantity` holds `number: SnaqNumber` and unit; `with_number(SnaqNumber, unit)` for explicit variance.
- **Units:** Default registry (`default_si_registry`) provides all 7 SI base units (m, kg, s, A, K, mol, cd) and all SI derived units with special names (J, C, V, F, ohm, S, Wb, T, H, Hz, N, Pa, W, lm, lx, Bq, Gy, Sv, kat), plus km, g, hour, minute, second, day, week, month, quarter, year, decade, century, millennium, mile, au, parsec, light_year, nautical_mile (nmi), joule, eV, celsius, fahrenheit (scale 5/9 for differences; absolute zero not modeled), Angle (rad, degree, arcmin, arcsec), **Area** (m2, are, hectare; prefix + m2 ŌåÆ (prefix┬Ęm)┬▓), **Volume** (m3, liter, milliliter; prefix + m3 ŌåÆ (prefix┬Ęm)┬│ via CUBED_LENGTH_REST), **Concentration** (molar, M), **Velocity** (knot), pressure (bar, atm, psi, mmHg, torr), mass (pound, ounce), energy (calorie, BTU, Wh, kWh), power (horsepower, hp), dimensionless (percent, ppm, ppb), solid angle (steradian, sr). Squared-length: prefix + "m2" or "m┬▓" in `try_lookup` resolves to (prefix┬Ęm)┬▓; cubed-length: CUBED_LENGTH_REST for km3, cm3, etc. Registry stores singular forms; plural input normalized via `singularize`. Long-form aliases via `add_derived_alias`. Prefix resolution via `get_unit_with_prefix`. Quantity has `full_simplify` and `full_simplify_with_registry`. **Symbol registry** default also has physical constants: c, h, hbar, R (dimensionless numeric values; use with units e.g. c * m / s).
- **Errors:** Runtime errors can carry an optional **span** (RunError::at, RunError.span). **Dimension labels in messages:** Dimension mismatch and incompatible-units messages show each unit via `Unit::dimension_label_for_error()` (scalar ŌåÆ "none", else unit display). When present and source is passed to format_run_error_with_source, output includes "at line X, column Y" and a snippet with squiggle (same style as parse errors). **Span propagation:** run()/run_with_registry() set ProgramDef::spanned_root so the expression graph gets spans; value() uses run_err_with_span(db, expr, kind) for direct errors and with_span_if_missing for errors from helpers; add_values, sub_values, mul_values, div_values, cmp_values, neg_value accept Option<Span> and use RunError::at when Some; eval_call/eval_if errors get call-site span via with_span_if_missing; map result collection and vector method branches (sum, mean, min, max, product, variance, stddev) attach the MethodCall span to helper errors via with_span_if_missing. CAS rewrite attaches the failing node's span to quantity_to_run_error and comparison errors. resolve_unit_expr and substitute_symbols_spanned_inner use RunError::at(span, ...) for UnknownUnit and SymbolHasNoValue. LALRPOP ParseError is mapped to crate::error::ParseError in the parser wrapper. Generated code has targeted `#[allow(...)]` for Clippy. Transpose on non-vector ŌåÆ `RunError::ExpectedVector`. Vector op length mismatch (element-wise or dot) ŌåÆ `RunError::VectorLengthMismatch { left_len, right_len }`. If condition not FuzzyBool ŌåÆ `RunError::ExpectedCondition`. If branches not blendable (boolean/vector) ŌåÆ `RunError::IfBranchTypeMismatch`. Tilde RHS Ōēż 0 or non-finite ŌåÆ `RunError::PrecisionMustBePositive`. to_quantity on Undefined ŌåÆ `RunError::UndefinedResult`. Vector reduction (mean, min, max, variance, stddev) on empty ŌåÆ `RunError::EmptyVectorReduction(method)`. Built-in invalid argument (e.g. sqrt of negative) ŌåÆ `RunError::InvalidArgument(msg)`.
- **┬▒Ōł×:** Only +Ōł× and -Ōł× supported (no NaN). Division: nonzero/0 ŌåÆ ┬▒Ōł×; 0/0 ŌåÆ `Value::Undefined` (not an error at Value level; run_numeric("0/0") ŌåÆ UndefinedResult). Infinite values have variance 0; display and arithmetic via f64. `Quantity::is_infinite()` / `is_finite()`; `convert_to` short-circuits when value is infinite (unit change only).
- **Value and symbols:** `Value` is `Numeric(Quantity)`, `Symbolic(SymbolicQuantity)`, **FuzzyBool(FuzzyBool)** (comparison result: True, False, or Uncertain(prob)), **Undefined** (empty block/program), `Vector(VectorValue)`, **`Function(UserFunction)`**, or **`BuiltinFunction(String)`** (built-in by name, e.g. sqrt; usable where a lambda is required). LitSymbol resolution: **scope first**, then **built-in function name** (if `functions::param_names(name).is_some()` ŌåÆ `Value::BuiltinFunction(name)`), then unit registry, then symbolic. StoredValue includes `BuiltinFunction(String)` so e.g. `f = sqrt` works. Display: Numeric/Symbolic as before; FuzzyBool as "true"/"false"/"uncertain(prob)". `to_quantity` on FuzzyBool or on Symbolic with comparison expr ŌåÆ `RunError::BooleanResult`. Comparisons use statistical P(A>B) and CONFIDENCE_THRESHOLD (0.95); when variance is zero or means equal (for Ge/Le) result is crisp. `SymbolRegistry` maps symbol names to numeric values (built-ins: pi, ŽĆ, e, sqrt_2, sqrt_3, phi, c, h, hbar, R). Bare identifiers resolve to `LitSymbol`; at eval time LitSymbol is resolved **scope first**, then unit registry, then symbolic. **run_numeric** uses `substitute_symbols(def, symbol_registry, unit_registry)`, which collects names bound in the program and leaves them as LitSymbol so `value()` resolves them from scope; other identifiers are substituted from symbol registry then unit registry. So variable bindings that shadow unit names affect both `run()` and `run_numeric()`. Evaluation returns `Value`. `SymbolicExpr` (Number, Symbol, Sum, Product, Neg, Div, Call, Eq, Ne, Lt, Le, Gt, Ge); `SymbolicQuantity(expr, unit)`. Comparison expressions: numeric comparison ŌåÆ `Value::FuzzyBool`; symbolic comparison stays Symbolic; `expr.is_comparison()` used to reject `to_quantity`. Default: symbolic result; `run_numeric()` on comparison returns `BooleanResult`. CLI: default symbolic, `--numeric` for numeric. Error `SymbolHasNoValue(name)` when substituting and symbol has no value.
- **Functions:** Built-ins in `functions.rs`: trig (sin, cos, tan); inverse trig (asin, acos, atan, atan2); hyperbolic (sinh, cosh, tanh, asinh, acosh, atanh); elementary (sqrt, cbrt, abs, sign, floor, ceil, round, trunc, mod); exp/log (exp, ln, log, log10, log2); pow, cbrt; probability (norm_cdf, norm_inv, factorial, binom, perm); gcd, lcm; max, min; take. Symbolic sugar: `^` ŌåÆ pow, `%` ŌåÆ mod, `!` postfix ŌåÆ factorial, `ŌłÜ` prefix ŌåÆ sqrt, `Ōłø` prefix ŌåÆ cbrt. Statistics: corr(X,Y) (eval_call special-case), V.median(), V.quantile(p). For well-known angles (0, ŽĆ/6, ŽĆ/4, ŽĆ/3, ŽĆ/2, ŽĆ and degree 0┬░, 30┬░, 45┬░, 60┬░, 90┬░, 180┬░), trig returns symbolic exact results (e.g. ŌłÜ2/2, ŌłÜ3/2 via constants sqrt_2, sqrt_3). `KnownAngle`, `known_angle_from_rad`, `known_angle_from_symbolic`, `exact_sin/cos/tan`; eval_call uses them so symbolic run keeps constants (ŽĆ, ŌłÜ2, ŌłÜ3) in display and run_numeric yields a number. Each built-in declares param names; call-site args bound positionally then by name. Unknown function ŌåÆ `RunError::UnknownFunction`. Wrong dimension for trig ŌåÆ `RunError::ExpectedAngle { actual }`. Symbolic calls: if any arg is symbolic and not a known angle, result is `Value::Symbolic(SymbolicExpr::Call(name, args))`. Variance is propagated through trig via first-order propagation.
- **User-defined functions:** Syntax `fn (params) => (expr)` or `=> { block }` (anonymous), `fn name(params) => ...` (named; FnName = Ident | FuncIdent so `fn f(a,b) => ...` parses). Params: `Ident` or `Ident = Expr` (default). IR: `ExprDef::Lambda`, `ExprDef::CallExpr`; `ExprData` same. Value: `Value::Function(UserFunctionId)`; scope: `StoredValue::Function(id)`. `user_function.rs`: `UserFunction` (params, body, closure_env), thread-local registry (register/get). **Call:** eval_call resolves name in scope first; if `StoredValue::Function(id)` ŌåÆ eval_user_call (bind positional, then named args, fill missing from defaults in closure scope, extend closure env, evaluate body); else built-in path. **Errors:** missing required arg, unknown parameter name (named arg not in params), duplicate parameter, too many positional args, not callable. **Cannot shadow built-ins:** eval_binding_chain checks `functions::param_names(name).is_some()` before extend ŌåÆ `RunError::CannotObfuscateBuiltin(name)`. CAS: Lambda/CallExpr in interner, convert, canonical, rewrite, substitute (recurse; substitute treats lambda param names as bound). Display: `Value::Function` ŌåÆ `"<function>"`; when program result is a function, run_format and CLI show `<function>` (run_numeric / to_quantity on function ŌåÆ BindingValueNotSupported). **CLI `-n`:** single evaluation via `run()` then branch (Function ŌåÆ print `<function>`, else `to_quantity` and format).
- **Conditionals (if/then/else):** Syntax `if condition then expression else expression`; condition and branches are full expressions. Condition must evaluate to **FuzzyBool**. **Crisp:** True ŌåÆ only then branch evaluated; False ŌåÆ only else branch. **Superposition (Uncertain(p)):** both branches evaluated and blended. Numeric: both quantities converted to common unit (`Quantity::smaller_unit`), then blended mean = p┬Ęmean_A + (1ŌłÆp)┬Ęmean_B and variance via Law of Total Variance (inner + between). Symbolic: weighted sum P┬Ęthen_expr + (1ŌłÆP)┬Ęelse_expr as SymbolicExpr (same dimension required). Type mismatch: both branches must be numeric or symbolic; FuzzyBool or Vector ŌåÆ `RunError::IfBranchTypeMismatch`. Condition not FuzzyBool ŌåÆ `RunError::ExpectedCondition`. CAS: `ExprNode::If(cond, then, else)`; rewrite constant-folds when condition is `LitFuzzyBool(True)` or `LitFuzzyBool(False)` to single branch.
- **CAS:** `cas/` module: interned AST (`ExprId`, `ExprNode`, `ExprInterner`), canonicalization (flatten Add/Mul, sort by rank), rewrite (identity/annihilation, constant fold, like-term collection). **LitFuzzyBool:** `ExprNode::LitFuzzyBool(FuzzyBool)`; rewrite constant-folds comparisons using statistical P(A>B) and CONFIDENCE_THRESHOLD to `LitFuzzyBool(True/False/Uncertain(prob))`. **If:** `ExprNode::If(ExprId, ExprId, ExprId)`; rewrite folds to then/else branch when condition is LitFuzzyBool(True/False). **WithPrecision:** `ExprNode::WithPrecision(ExprId, ExprId)`; pass-through in canonicalization and rewrite. **Block:** `ExprNode::Block(Vec<ExprId>)`; order preserved in canonicalize and rewrite (recurse into each child). **Comparison nodes:** `ExprNode::Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two ExprId); pass-through in canonicalization; rewrite when both children Lit ŌåÆ `LitFuzzyBool`. **Mul operands:** sorted by rank except when both operands are vector-valued (`VecLiteral` or `Transpose`): then order is preserved so row├Ścolumn (dot) vs column├Śrow (outer) is not swapped. `ExprNode::VecLiteral(Vec<ExprId>)` pass-through. `simplify_symbolic(def, registry)` and `simplify_numeric(def, unit_registry, symbol_registry)`; `substitute_symbols(def, symbol_registry, unit_registry)` substitutes LitSymbol from symbol registry (scalar) or unit registry (1 in that unit), then CAS; handles `LitFuzzyBool` and `WithPrecision`. Run pipelines apply CAS before building Salsa program. Division in rewrite: nonzero/0 constant-folds to ┬▒Ōł×; 0/0 left as Div so value() maps it to `Value::Undefined`.
- **Dates (temporal literals):** Syntax `@YYYY`, `@YYYY-MM`, `@YYYY-MM-DD`, `@YYYY-MM-DDTHH`, `@YYYY-MM-DDTHH:MM`, `@YYYY-MM-DDTHH:MM:SS` (ISO 8601, UTC). Lexer: `Tok::TemporalLiteral(String)`, `take_temporal_literal`; invalid/incomplete ŌåÆ `LexicalError::InvalidTemporal`. IR: `LitTemporal(String)` at parse; resolve ŌåÆ `LitDate(GranularDate)` via `date::parse_temporal_literal` (grain inference, anchor in Unix seconds). `GranularDate { anchor_epoch_sec, grain }` in `date.rs`; `min_epoch_sec`/`max_epoch_sec` for interval bounds. Value/Scope: `Value::Date(GranularDate)`, `StoredValue::Date`; CAS LitDate pass-through. Algebra: Date ┬▒ Time (duration must have Time dimension; duration grain must not be finer than date grain ŌåÆ `IncompatibleDateGrain`); Date ŌłÆ Date ŌåÆ seconds (Quantity). Comparison: Date vs Date ŌåÆ interval overlap ŌåÆ `FuzzyBool` (true/false/uncertain(0.5)); Date vs non-Date ŌåÆ `InvalidArgument`. Single-comparison RHS in grammar is `AddExpr` (not `Expr`) so e.g. `@2025 < @2026` parses. Docs: docs/DATES.md.
- **Maps:** Map literals `{ key: value, ... }` (at least one entry; keys unquoted idents). Empty `{}` is a block. **Access:** `.key` (Member) and `[key]` (Index); bracket content is the key string (trimmed). Missing key ŌåÆ `Value::Undefined`. **Storage:** `map_registry` is **shared** (Arc<Mutex<RegistryStore>>) so stream feeders (other threads) can register row-as-map and main thread can get_key; `Value::Map(u64)`, `StoredValue::Map(MapId)`; scope from_value/to_value. **Record/stream:** `Record` = Vec<(String, Value)>; `record_to_value(entries)` and `record_to_chunk_element(entries)` for hosts building tabular stream chunks. IR: `MapLiteral(Vec<(String, Box<ExprDef>)>)`; CAS (interner, convert, canonical, rewrite, substitute) recurse on value exprs. Member/Index on Value::Map use map_registry::get_key. Display: `<map>`; to_quantity on map ŌåÆ error. Bracket key: lexer emits BracketKey(trimmed) when `[` follows postfix factor; vector index interprets key as non-negative int or variable.
- **External stream inputs ($name):** Programs can refer to external data via `$name`. If the handle is missing at **consume** time (vector_into_stream), the stream yields one **StreamInputNotAvailable** error then completes (OneErrorStream). Lexer: `DollarIdent(name)`; grammar Factor ŌåÆ `ExternalStream(name)`; IR `ExprDef`/`ExprData`/`SpannedExprDefKind::ExternalStream`; resolve and CAS pass-through. Salsa input **StreamInputRegistry** (name ŌåÆ `StreamHandleId`); thread-local **stream handle registry** maps id ŌåÆ receiver (`futures::channel::mpsc::UnboundedReceiver<Chunk>`). **Chunk** = `Vec<Result<Option<Value>, RunError>>`; Host pushes chunks, drops sender for EOF. **create_stream_input()** in stream_handle returns (StreamHandleId, UnboundedSender<Chunk>) so Host can create and retain sender. `value()` for ExternalStream: read registry, return `Value::Vector(LazyVector::FromInput(handle_id))` or `UnboundStreamInput`. **LazyVector::FromInput(StreamHandleId)**; `vector_into_stream` takes receiver via `take_receiver(id)`, wraps in **ChunkFlattenStream** (flattens chunks to elements). Host: create channel (or create_stream_input), `register(receiver)` ŌåÆ id, set `StreamInputRegistry` with nameŌåÆid, `run_with_stream_inputs` ŌåÆ (Value, db), then `vector_into_stream` on vector result while pushing chunks.
