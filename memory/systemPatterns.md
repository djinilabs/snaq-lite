# System patterns

- **Parsing:** **Program/Blocks:** Top-level is a **Program** = expression list (newline or `;` separated; blank lines allowed) wrapped in `ExprDef::Block`. Lexer: `LBrace` `{`, `RBrace` `}`, `Semicolon` `;`, `Newline` (emit on `\n` or `\r\n`); `skip_whitespace` skips only space and tab. Grammar: `Program` = `ExprList` => `Block(list)`; `ExprList` empty or `Expr ExprListRest`; `ExprListRest` can consume separator-only (blank line). **Factor** includes `"{" ExprList "}"` => `Block(list)`. Parser entry: `ProgramParser` (parse returns one `ExprDef`, the top-level Block). Custom lexer in `lexer.rs` emits `Num(NumLiteral)` (raw string + value for implicit variance), `FuncIdent(name)` when identifier is followed by `(`, else `Ident(name)`; disambiguates `sin(1)` (call) vs `sin` (symbol). Keywords: "per" (division), "as" (unit conversion; boundary check so "aspect" stays Ident), **"if", "then", "else"** (conditionals). Tokens: `~` (Tilde, explicit precision). Comparison tokens: `==`, `!=`, `<`, `<=`, `>`, `>=` (two-char before single `<`/`>`). LALRPOP grammar in `expr.lalrpop` uses extern tokens (Num, Ident, FuncIdent, etc.). Precedence: top-level Expr (comparison) ← AddExpr (+, -, as) ← Term (*, /, per, implicit mul) ← **TildeBound** (~) ← PostfixFactor (transpose) ← Factor (unary minus, NumIdents, literal, `FuncIdent "(" ArgList ")"`, Ident, parentheses). **UnitTerm:** unit-only expression (IdentIdents, *, /, per) for RHS of "as" (e.g. "m", "meters per second"). **NumIdents:** number followed by one or more idents is one factor (e.g. "1 s", "2 pi rad"); parses as left-associated Mul chain so " / 1 s" divides by 1 s and "2 pi rad" has pi→symbol, rad→unit at resolve. **IdentIdents:** one or more idents with no number (e.g. "pi", "pi rad") is one factor; "pi rad" parses as Mul(pi, rad), resolve attributes to symbol and unit. **PiIdents:** "π" alone or "π" + one or more idents (e.g. "π rad") so unicode π behaves like "pi rad". Implicit mul RHS is only number or parenthesized expr (not ident). Call args: `CallArg` = `Ident ":" Expr` (named) or `Expr` (positional); `ArgList` left-recursive.
- **Variable bindings:** **Immutable** and **lexically scoped**. Syntax: `Ident = Expr` in block or top-level (single `=`, not `==`). Scope is an **interned** Salsa type wrapping a **persistent map** (`im::OrdMap`); extension is non-destructive so (scope, expr) is the memoization key and changing a root variable produces a new scope and invalidates only dependents. **StoredValue** (Numeric, FuzzyBool, Undefined; no Vector/Symbolic in v1) is Eq+Hash for the map. **Evaluation:** `value(db, scope, expr)`; LitSymbol → **scope first**, then unit registry (1 in that unit), then symbolic (so variables **shadow** unit names, e.g. DEF=3; DEF+2 → 5). Block → iterate items, Binding extends scope, last expression is result; block of only bindings → last binding’s RHS. Resolve: bare identifiers (LitUnit) → LitSymbol (no unit lookup at resolve). Resolve and CAS (interner, convert, canonical, rewrite, substitute) handle `ExprDef::Binding` / `ExprNode::Binding` (pass-through). Lexer: `Assign` token (single `=`). Grammar: `BlockItem` = `Ident "=" BlockItem` | `Expr` (chained assignment); `ExprList` uses `BlockItem`.
- **IR:** `ExprDef` has parse-time variants `LitScalar(NumLiteral)`, `LitWithUnit(NumLiteral, String)`, `LitUnit`, `Neg(Box<ExprDef>)`, `VecLiteral(Vec<ExprDef>)`, `Transpose(Box<ExprDef>)` and resolved `Lit(Quantity)`, `LitFuzzyBool(FuzzyBool)` (constant-folded comparison), `LitSymbol(String)`, `Call(String, Vec<CallArg>)`, `As(Box<ExprDef>, Box<ExprDef>)` (unit conversion: left value expr, right unit-only expr), **`If(Box<ExprDef>, Box<ExprDef>, Box<ExprDef>)`** (condition, then_branch, else_branch), **`WithPrecision(Box<ExprDef>, Box<ExprDef>)`** (left ~ right: explicit variance from RHS value), **`Block(Vec<ExprDef>)`** (sequence of expressions; value = last or undefined if empty), **`Binding(String, Box<ExprDef>)`** (variable binding in block context). Binary `Add`, `Sub`, `Mul`, `Div`. Comparison: `Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two `Box<ExprDef>`); **result is Value::FuzzyBool**. `CallArg` = `Positional(Box<ExprDef>)` or `Named(String, Box<ExprDef>)`. Two-step flow: parse → resolve (with unit registry) → `ExprDef` with `Lit(Quantity)`, `LitFuzzyBool`, `LitSymbol`, `Call`, `As`, `VecLiteral`, `If`, or `WithPrecision` for literals. RHS of `As` resolved via `resolve_unit_expr` (only LitUnit/Mul/Div → Lit(Quantity), no symbols). `ExprData` has `LitFuzzyBool(FuzzyBool)`, `Call(name, Vec<(Option<String>, Expression)>)`, `As(Expression, Expression)`, `VecLiteral(Vec<Expression>)`, `Transpose(Expression)`, **`WithPrecision(Expression, Expression)`**, and **`If(Expression, Expression, Expression)`**, **`Block(Vec<Expression>)`**, **`Binding(String, Expression)`**; `value(db, scope, expr)` returns `Result<Value, RunError>`; scope is interned, empty via `empty_scope(db)`. Block: evaluate expressions in order; return last value or `Value::Undefined` if empty.
- **Numbers and quantities:** Numeric values are `SnaqNumber` (value + variance). Variance propagates through arithmetic (Add/Sub: sum of variances; Mul/Div: standard propagation; scale by f64: variance × factor²). **Literal variance:** Lexer emits `NumLiteral { raw, value }`; `LitScalar`/`LitWithUnit` carry `NumLiteral`. Resolve uses `implicit_variance()`: no decimal point → abs_err 0.5, variance 0.25; N decimal places (mantissa only) → abs_err = 5×10^−(N+1), variance = abs_err². **Explicit precision:** `expr ~ expr` (tilde) → `WithPrecision(l, r)`; precedence above * / + −. Evaluation: result = left value, variance = (right.value())²; right's variance discarded; RHS must be > 0 → `RunError::PrecisionMustBePositive`. `Quantity` holds `number: SnaqNumber` and unit; `with_number(SnaqNumber, unit)` for explicit variance.
- **Units:** Default registry (`default_si_registry`) provides all 7 SI base units (m, kg, s, A, K, mol, cd) and all SI derived units with special names (J, C, V, F, ohm, S, Wb, T, H, Hz, N, Pa, W, lm, lx, Bq, Gy, Sv, kat), plus km, g, hour, minute, second, day, week, month, quarter, year, decade, century, millennium, mile, au, parsec, light_year, joule, eV, celsius, Angle (rad base, degree = π/180 rad), and **Area** (m2, are = 100 m², hectare = 100 are; hectare via prefix "hect" + "are"). Squared-length: prefix + "m2" or "m²" in `try_lookup` resolves to (prefix·m)² so km2, cm2, mm2 convert correctly. Area aliases: m², sqm, squaremeter, squarekilometer, etc.; inch (in), in2, in², sqin, squareinch; foot (ft), ft2, sqft, squarefoot, square_foot. Registry stores singular forms; plural input normalized via `singularize`. Long-form aliases via `add_derived_alias`. Prefix resolution via `get_unit_with_prefix`. Quantity has `full_simplify` and `full_simplify_with_registry`.
- **Errors:** Runtime errors can carry an optional **span** (RunError::at, RunError.span). **Dimension labels in messages:** Dimension mismatch and incompatible-units messages show each unit via `Unit::dimension_label_for_error()` (scalar → "none", else unit display). When present and source is passed to format_run_error_with_source, output includes "at line X, column Y" and a snippet with squiggle (same style as parse errors). **Span propagation:** run()/run_with_registry() set ProgramDef::spanned_root so the expression graph gets spans; value() uses run_err_with_span(db, expr, kind) for direct errors and with_span_if_missing for errors from helpers; add_values, sub_values, mul_values, div_values, cmp_values, neg_value accept Option<Span> and use RunError::at when Some; eval_call/eval_if errors get call-site span via with_span_if_missing; map result collection and vector method branches (sum, mean, min, max, product, variance, stddev) attach the MethodCall span to helper errors via with_span_if_missing. CAS rewrite attaches the failing node's span to quantity_to_run_error and comparison errors. resolve_unit_expr and substitute_symbols_spanned_inner use RunError::at(span, ...) for UnknownUnit and SymbolHasNoValue. LALRPOP ParseError is mapped to crate::error::ParseError in the parser wrapper. Generated code has targeted `#[allow(...)]` for Clippy. Transpose on non-vector → `RunError::ExpectedVector`. Vector op length mismatch (element-wise or dot) → `RunError::VectorLengthMismatch { left_len, right_len }`. If condition not FuzzyBool → `RunError::ExpectedCondition`. If branches not blendable (boolean/vector) → `RunError::IfBranchTypeMismatch`. Tilde RHS ≤ 0 or non-finite → `RunError::PrecisionMustBePositive`. to_quantity on Undefined → `RunError::UndefinedResult`. Vector reduction (mean, min, max, variance, stddev) on empty → `RunError::EmptyVectorReduction(method)`. Built-in invalid argument (e.g. sqrt of negative) → `RunError::InvalidArgument(msg)`.
- **±∞:** Only +∞ and -∞ supported (no NaN). Division: nonzero/0 → ±∞; 0/0 → `Value::Undefined` (not an error at Value level; run_numeric("0/0") → UndefinedResult). Infinite values have variance 0; display and arithmetic via f64. `Quantity::is_infinite()` / `is_finite()`; `convert_to` short-circuits when value is infinite (unit change only).
- **Value and symbols:** `Value` is `Numeric(Quantity)`, `Symbolic(SymbolicQuantity)`, **FuzzyBool(FuzzyBool)** (comparison result: True, False, or Uncertain(prob)), **Undefined** (empty block/program), `Vector(VectorValue)`, **`Function(UserFunction)`**, or **`BuiltinFunction(String)`** (built-in by name, e.g. sqrt; usable where a lambda is required). LitSymbol resolution: **scope first**, then **built-in function name** (if `functions::param_names(name).is_some()` → `Value::BuiltinFunction(name)`), then unit registry, then symbolic. StoredValue includes `BuiltinFunction(String)` so e.g. `f = sqrt` works. Display: Numeric/Symbolic as before; FuzzyBool as "true"/"false"/"uncertain(prob)". `to_quantity` on FuzzyBool or on Symbolic with comparison expr → `RunError::BooleanResult`. Comparisons use statistical P(A>B) and CONFIDENCE_THRESHOLD (0.95); when variance is zero or means equal (for Ge/Le) result is crisp. `SymbolRegistry` maps symbol names to numeric values (built-ins: pi, π, e, sqrt_2, sqrt_3). Bare identifiers resolve to `LitSymbol`; at eval time LitSymbol is resolved **scope first**, then unit registry, then symbolic. **run_numeric** uses `substitute_symbols(def, symbol_registry, unit_registry)`, which collects names bound in the program and leaves them as LitSymbol so `value()` resolves them from scope; other identifiers are substituted from symbol registry then unit registry. So variable bindings that shadow unit names affect both `run()` and `run_numeric()`. Evaluation returns `Value`. `SymbolicExpr` (Number, Symbol, Sum, Product, Neg, Div, Call, Eq, Ne, Lt, Le, Gt, Ge); `SymbolicQuantity(expr, unit)`. Comparison expressions: numeric comparison → `Value::FuzzyBool`; symbolic comparison stays Symbolic; `expr.is_comparison()` used to reject `to_quantity`. Default: symbolic result; `run_numeric()` on comparison returns `BooleanResult`. CLI: default symbolic, `--numeric` for numeric. Error `SymbolHasNoValue(name)` when substituting and symbol has no value.
- **Functions:** Built-ins in `functions.rs`: trig (sin, cos, tan); inverse trig (asin, acos, atan, atan2); hyperbolic (sinh, cosh, tanh, asinh, acosh, atanh); elementary (sqrt, cbrt, abs, sign, floor, ceil, round, trunc, mod); exp/log (exp, ln, log, log10, log2); pow, cbrt; probability (norm_cdf, norm_inv, factorial, binom, perm); gcd, lcm; max, min; take. Symbolic sugar: `^` → pow, `%` → mod, `!` postfix → factorial, `√` prefix → sqrt, `∛` prefix → cbrt. Statistics: corr(X,Y) (eval_call special-case), V.median(), V.quantile(p). For well-known angles (0, π/6, π/4, π/3, π/2, π and degree 0°, 30°, 45°, 60°, 90°, 180°), trig returns symbolic exact results (e.g. √2/2, √3/2 via constants sqrt_2, sqrt_3). `KnownAngle`, `known_angle_from_rad`, `known_angle_from_symbolic`, `exact_sin/cos/tan`; eval_call uses them so symbolic run keeps constants (π, √2, √3) in display and run_numeric yields a number. Each built-in declares param names; call-site args bound positionally then by name. Unknown function → `RunError::UnknownFunction`. Wrong dimension for trig → `RunError::ExpectedAngle { actual }`. Symbolic calls: if any arg is symbolic and not a known angle, result is `Value::Symbolic(SymbolicExpr::Call(name, args))`. Variance is propagated through trig via first-order propagation.
- **User-defined functions:** Syntax `fn (params) => (expr)` or `=> { block }` (anonymous), `fn name(params) => ...` (named; FnName = Ident | FuncIdent so `fn f(a,b) => ...` parses). Params: `Ident` or `Ident = Expr` (default). IR: `ExprDef::Lambda`, `ExprDef::CallExpr`; `ExprData` same. Value: `Value::Function(UserFunctionId)`; scope: `StoredValue::Function(id)`. `user_function.rs`: `UserFunction` (params, body, closure_env), thread-local registry (register/get). **Call:** eval_call resolves name in scope first; if `StoredValue::Function(id)` → eval_user_call (bind positional, then named args, fill missing from defaults in closure scope, extend closure env, evaluate body); else built-in path. **Errors:** missing required arg, unknown parameter name (named arg not in params), duplicate parameter, too many positional args, not callable. **Cannot shadow built-ins:** eval_binding_chain checks `functions::param_names(name).is_some()` before extend → `RunError::CannotObfuscateBuiltin(name)`. CAS: Lambda/CallExpr in interner, convert, canonical, rewrite, substitute (recurse; substitute treats lambda param names as bound). Display: `Value::Function` → `"<function>"`; when program result is a function, run_format and CLI show `<function>` (run_numeric / to_quantity on function → BindingValueNotSupported). **CLI `-n`:** single evaluation via `run()` then branch (Function → print `<function>`, else `to_quantity` and format).
- **Conditionals (if/then/else):** Syntax `if condition then expression else expression`; condition and branches are full expressions. Condition must evaluate to **FuzzyBool**. **Crisp:** True → only then branch evaluated; False → only else branch. **Superposition (Uncertain(p)):** both branches evaluated and blended. Numeric: both quantities converted to common unit (`Quantity::smaller_unit`), then blended mean = p·mean_A + (1−p)·mean_B and variance via Law of Total Variance (inner + between). Symbolic: weighted sum P·then_expr + (1−P)·else_expr as SymbolicExpr (same dimension required). Type mismatch: both branches must be numeric or symbolic; FuzzyBool or Vector → `RunError::IfBranchTypeMismatch`. Condition not FuzzyBool → `RunError::ExpectedCondition`. CAS: `ExprNode::If(cond, then, else)`; rewrite constant-folds when condition is `LitFuzzyBool(True)` or `LitFuzzyBool(False)` to single branch.
- **CAS:** `cas/` module: interned AST (`ExprId`, `ExprNode`, `ExprInterner`), canonicalization (flatten Add/Mul, sort by rank), rewrite (identity/annihilation, constant fold, like-term collection). **LitFuzzyBool:** `ExprNode::LitFuzzyBool(FuzzyBool)`; rewrite constant-folds comparisons using statistical P(A>B) and CONFIDENCE_THRESHOLD to `LitFuzzyBool(True/False/Uncertain(prob))`. **If:** `ExprNode::If(ExprId, ExprId, ExprId)`; rewrite folds to then/else branch when condition is LitFuzzyBool(True/False). **WithPrecision:** `ExprNode::WithPrecision(ExprId, ExprId)`; pass-through in canonicalization and rewrite. **Block:** `ExprNode::Block(Vec<ExprId>)`; order preserved in canonicalize and rewrite (recurse into each child). **Comparison nodes:** `ExprNode::Eq`, `Ne`, `Lt`, `Le`, `Gt`, `Ge` (each two ExprId); pass-through in canonicalization; rewrite when both children Lit → `LitFuzzyBool`. **Mul operands:** sorted by rank except when both operands are vector-valued (`VecLiteral` or `Transpose`): then order is preserved so row×column (dot) vs column×row (outer) is not swapped. `ExprNode::VecLiteral(Vec<ExprId>)` pass-through. `simplify_symbolic(def, registry)` and `simplify_numeric(def, unit_registry, symbol_registry)`; `substitute_symbols(def, symbol_registry, unit_registry)` substitutes LitSymbol from symbol registry (scalar) or unit registry (1 in that unit), then CAS; handles `LitFuzzyBool` and `WithPrecision`. Run pipelines apply CAS before building Salsa program. Division in rewrite: nonzero/0 constant-folds to ±∞; 0/0 left as Div so value() maps it to `Value::Undefined`.
- **Vectors:** `Value::Vector(VectorValue)` where `VectorValue { inner: LazyVector, orientation: VectorOrientation }`. Orientation: **Column** (default for literals and 2D column yields), **Row** (after transpose); transpose flips orientation. Lazy vector: `LazyVector::FromExprs`, `FromEvaluated`, `Map`, `ZipMap`, `Outer`, stub `Transform`, **Transpose** (postfix `'`), and **Take** (slice: source, start, length): 1D stream = identity; 2D = `VectorStream::Transposed` yields columns; Take = `TakeVectorStream` skip then yield. **Vector–vector ops** (add, sub, mul, div, and comparison ==, !=, <, <=, >, >=) dispatch by orientation: **column×column** or **row×row** → element-wise (lazy `ZipMap`), result column or row (comparisons yield FuzzyBool per element); **column×row** → outer product (lazy `Outer`), result = vector of column vectors (matrix of FuzzyBool for comparisons); **row×column** → dot product (mul), sum(left) op sum(right) (add/sub), or compare(sum(left), sum(right)) (comparisons), scalar. `VectorBinaryOp` (Add, Sub, Mul, Div, Eq, Ne, Lt, Le, Gt, Ge) and `VectorMapOp` (including Eq/Ne/Lt/Le/Gt/Ge(Box<Value>) for vector op scalar, and **UserMap(UserFunctionId)** for `V.map(fn)`) used by ZipMap, Outer, and Map. Streams: `ZippedVectorStream`, `OuterProductVectorStream`, `TakeVectorStream`. **Member access:** `V.length` → `ExprDef::Member` (property, eval: collect stream and return count); `V.map(fn)` (fn = user function or single-arg built-in e.g. sqrt, sin; BuiltinFunction applied via apply_unary_builtin per element), `V.take(start, length)`, `V.sum()`, `V.mean()`, `V.min()`, `V.max()`, `V.dot(other)`, `V.norm()`, `V.product()`, `V.variance()`, `V.stddev()`, `V.all()`, `V.any()` → `ExprDef::MethodCall` (method, eval: dispatch on name). Lexer emits `MethodIdent` after `.` when ident is followed by `(` so `.map(` is unambiguous from `.length`; empty `ArgList` allowed (e.g. `V.sum()`). **Implicit mapping:** vector ±/×/÷ scalar → Map; vector op scalar comparison → Map (0/1 per element); neg/sin/cos/tan(vector) → Map. **Slicing and indexing:** Built-in `take(vector, start, length)` (eval_call special-case, not call_builtin) returns `Vector(LazyVector::Take)`; index access `V[expr]` or `V.0` → `ExprDef::Index` (grammar PostfixFactor "[" Expr "]" or "." Num); lexer Dot only when `.` follows non-number (`last_was_number` and `any_token_emitted`); value() for Index: base vector, index non-negative integer (InvalidIndex if not), Take(_, index, 1), collect single element or IndexOutOfBounds. `vector_into_stream` handles Map (with db for UserMap), ZipMap, Outer, Take; op applied via `apply_map_op` / `apply_binary_op`. to_quantity on vector → `UnsupportedVectorOperation`. Literal syntax `[ expr, ... ]` or `[]`; postfix `'` on any Factor. WASM-compatible (futures only in lang crate).
